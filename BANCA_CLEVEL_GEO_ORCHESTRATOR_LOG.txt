**Arquivo**: `src/observability/distributed_tracer.py` (novo)

**Integração Principal**: 
- `src/core/orchestrator.py` → função `execute_waves()` (linha ~847)

# Melhoria: Observabilidade Distribuída com Tracing Automático

**Arquivo:** `geo_orchestrator/observability/distributed_tracer.py`

**Problema:** Seu orquestrador executa tarefas em paralelo com múltiplos LLMs, mas carece de visibilidade em tempo real sobre gargalos, latências por modelo e falhas em waves específicas. Checkpoints existentes não capturam contexto de execução distribuída.[2][4]

**Solução Concreta:** Implementar integração com **OpenTelemetry + Jaeger** para rastreamento automático de:

1. **Span por tarefa tipada**: Cada decomposição gera span rastreando entrada → modelo selecionado → latência → output
2. **Correlação de waves**: Links entre dependências, identificando qual wave bloqueou parallelização
3. **Métricas por modelo**: Scoring adaptativo alimentado por histograma de latências reais (Claude Opus vs GPT-4o vs Gemini Flash)
4. **Budget tracking visual**: Cada token consumido rastreado por modelo, com alertas quando aproximar limites

**Integração na CLI:**

```python
# geo_orchestrator/cli.py - adicionar flag
@click.command()
@click.option('--trace-backend', default='jaeger', help='jaeger|zipkin')
@click.option('--enable-metrics', is_flag=True)
def run(trace_backend, enable_metrics):
    from geo_orchestrator.observability.distributed_tracer import setup_tracing
    setup_tracing(backend=trace_backend, enable_metrics=enable_metrics)
    orchestrator.execute_with_tracing()
```

**Benefício Imediato:** Você identifica se Gemini Flash está lento numa wave específica e o router adaptativo aprende automaticamente para próximas demandas.[1][5] Isto reduz debugging de horas para minutos quando performance degrada.

**Por que agora:** Em 2026, multi-LLM em produção exige observabilidade nativa.[4] Seu sistema já tem infraestrutura (cache SHA-256, checkpoints, quality gates) — tracing fecha o loop de visibilidade operacional sem overhead significativo (OpenTelemetry é assíncrono).

# Melhoria: Observabilidade em Tempo Real com Traces Distribuídos

## Proposta Concreta

**Arquivo**: `src/observability/tracer.py` (novo)

**Integração**: Implementar OpenTelemetry com exportador Jaeger/Datadog nas funções críticas do orquestrador.

## Problema Identificado

Com 4.738 linhas distribuindo entre 4 LLMs em paralelo com waves/dependencies, há **cegueira operacional**:
- Não há visibilidade de latência por modelo
- Impossível debugar gargalos em cascata de tarefas
- Falhas silenciosas em checkpoints
- Budget guard não correlaciona consumo com performance

## Solução

```python
# src/observabilidade/tracer.py
from opentelemetry import trace, metrics
from opentelemetry.exporter.jaeger.thrift import JaegerExporter
from opentelemetry.sdk.trace import TracerProvider
from opentelemetry.sdk.trace.export import BatchSpanProcessor

class GeoOrchTracer:
    def __init__(self, service_name="geo-orchestrator"):
        jaeger_exporter = JaegerExporter(
            agent_host_name="localhost",
            agent_port=6831,
        )
        trace.set_tracer_provider(
            TracerProvider(
                resource=Resource.create({"service.name": service_name})
            )
        )
        trace.get_tracer_provider().add_span_processor(
            BatchSpanProcessor(jaeger_exporter)
        )
        self.tracer = trace.get_tracer(__name__)
    
    def trace_task_execution(self, task_id, model_name):
        """Context manager para cada tarefa distribuída"""
        return self.tracer.start_as_current_span(
            f"task.{task_id}.{model_name}",
            attributes={
                "model": model_name,
                "task.id": task_id,
                "wave.priority": "auto"
            }
        )
```

**Integração em** `src/router/adaptive_router.py`:
```python
with tracer.trace_task_execution(task.id, selected_model):
    result = await distribute_to_model(task, selected_model)
    # Jaeger correlaciona: latência, modelo, custo, gate pass/fail
```

## Benefícios Imediatos

1. **Debug**: Cascata de waves visível em gráfico Gantt
2. **Otimização**: Identifica modelo mais lento por tipo de tarefa
3. **Budget**: Correlaciona custo com latência real
4. **SLA**: Detecta regressão antes de produção

**CLI aprimorado**: `python cli.py run --trace=jaeger --dashboard=localhost:16686`

Mínimo overhead (~2% latência), máximo insight operacional.
