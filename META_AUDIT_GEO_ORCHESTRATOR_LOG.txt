### 2. **Tratamento Incompleto de Exceções - Linha Inacessível**
**Arquivo:** `src/llm_client.py`, linhas 49-52

```python
        # Should never reach here, but satisfy type checker
        raise last_error  # type: ignore[misc]
```

**Problema:** 
- Se `MAX_RETRIES=1`, o loop executa 2 iterações. Se ambas falham e a segunda não é retentável, `last_error` é None.
- Ignora tipo (`# type: ignore[misc]`) esconde bug real.
- Lógica de retry está quebrada: nunca atinge o `raise last_error`.

**Correção:**
```python
async def query(
    self,
    prompt: str,
    system: str = "",
    max_tokens: int = 4000,
) -> LLMResponse:
    """Send a prompt to the configured LLM and return a unified response."""
    last_error: Exception | None = None

    for attempt in range(1 + self.MAX_RETRIES):
        try:
            return await self._call(prompt, system, max_tokens)
        except (httpx.HTTPStatusError, httpx.TimeoutException) as exc:
            last_error = exc
            retryable = (
                (isinstance(exc, httpx.HTTPStatusError) and 
                 exc.response.status_code in (429, 500)) or
                isinstance(exc, httpx.TimeoutException)
            )
            if retryable and attempt < self.MAX_RETRIES:
                await asyncio.sleep(self.RETRY_DELAY)
                continue
            # Não tentaremos mais, sai do loop
            break

    # Garante que sempre há erro para lançar
    if last_error is None:
        raise RuntimeError("Unexpected state: no error captured in retry loop")
    raise last_error
```

- **Agrupamento de requests:** Combine múltiplas consultas em um único prompt para reduzir RPM e otimizar TPM[3].

- **Distribua carga:** Use múltiplos projetos ou proxies, mas evite abuso para não violar termos[1][3].
- **Pré-processamento:** Cache respostas comuns e use context caching (pago: US$0.025/1M tokens)[6].

### Implementação correta de exponential backoff
Para tratar 429, use backoff exponencial com jitter para evitar thundering herd. Exemplo em Python (usando `tenacity` ou nativo):

```python
import time
import random
from requests.exceptions import HTTPError

def make_request_with_backoff(url, max_retries=5):
    retry_delay = 1  # Base delay em segundos
    for attempt in range(max_retries):
        try:
            response = requests.post(url)  # Sua chamada à API Gemini
            response.raise_for_status()
            return response.json()
        except HTTPError as e:
            if e.response.status_code == 429:
                # Exponential backoff: 2^attempt + jitter (0-1s)
                delay = (2 ** attempt) * retry_delay + random.uniform(0, 1)
                print(f"429 detectado. Aguardando {delay:.2f}s (tentativa {attempt+1}/{max_retries})")
                time.sleep(delay)
            else:
                raise
    raise Exception("Máximo de retentativas excedido")
```

- **Parâmetros ideais:** Inicie com 1s, multiplique por 2 por tentativa, adicione jitter randômico (0-1s). Máximo 5-8 retries. Monitore `retry-after` header se presente[3].
- Teste em produção para ajustar, pois limites variam por modelo e tier[2][3].

# Análise e Correção Robusta para Gemini (e outros LLMs) - 429 Errors

## Problema Identificado

```
Free Tier Gemini: 15 RPM (1 request a cada 4 segundos)
Código atual: retry simples, sem rate limiting local, sem fila
Resultado: múltiplas requisições em paralelo -> 429 errors
```

## Solução Completa com 3 Camadas

### 1️⃣ **Rate Limiter Local (Token Bucket + Per-Provider)**

```python
# rate_limiter.py
"""Rate limiter com estratégia token bucket por provider."""

from __future__ import annotations

import asyncio
import time
from dataclasses import dataclass
from enum import Enum


class Provider(str, Enum):
    ANTHROPIC = "anthropic"
    OPENAI = "openai"
    GOOGLE = "google"
    PERPLEXITY = "perplexity"


@dataclass
class RateLimitConfig:
    """Configuração de rate limit por provider."""
    requests_per_minute: int
    burst_size: int = 2  # Permite 2 requests seguidos, depois respeita RPM


class TokenBucketLimiter:
    """Token bucket algorithm - permite rajadas controladas."""

    def __init__(self, config: RateLimitConfig) -> None:
        self.rpm = config.requests_per_minute
        self.burst_size = config.burst_size
        self.tokens = float(config.burst_size)  # Começa com burst cheio
        self.last_update = time.time()
        self.lock = asyncio.Lock()
        
        # Tokens adicionados por segundo: RPM / 60
        self.refill_rate = config.requests_per_minute / 60.0

    async def acquire(self) -> None:
        """Espera até ter 1 token disponível."""
        async with self.lock:
            while self.tokens < 1.0:
                # Calcula quanto tempo esperar
                now = time.time()
                elapsed = now - self.last_update
                self.tokens += elapsed * self.refill_rate
                
                if self.tokens < 1.0:
                    # Quanto tempo falta para 1 token?
                    wait_time = (1.0 - self.tokens) / self.refill_rate
                    await asyncio.sleep(wait_time)
                    continue
                
                self.last_update = now
                break
            
            # Consome 1 token
            self.tokens -= 1.0
            self.last_update = time.time()

    def get_status(self) -> dict:
        """Debug: retorna estado atual."""
        now = time.time()
        elapsed = now - self.last_update
        current_tokens = min(
            self.tokens + elapsed * self.refill_rate,
            float(self.burst_size)
        )
        return {
            "tokens": round(current_tokens, 2),
            "max_burst": self.burst_size,
            "rpm": self.rpm,
        }


# Configurações por provider (free tier oficial)
RATE_LIMIT_CONFIGS = {
    Provider.GOOGLE: RateLimitConfig(requests_per_minute=15, burst_size=1),
    Provider.OPENAI: RateLimitConfig(requests_per_minute=3500, burst_size=10),
    Provider.ANTHROPIC: RateLimitConfig(requests_per_minute=50, burst_size=3),
    Provider.PERPLEXITY: RateLimitConfig(requests_per_minute=20, burst_size=2),
}
```
